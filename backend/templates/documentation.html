User Management Endpoints

Register a new user

POST https://api-docs-studyhacks.onrender.com/register

                    
                        ## Register User

                        Registers a new user in the system.
                        
                        - **URL**: `/register`
                        - **Method**: `POST`
                        
                        ### Request Body
                        
                        - `email` (string, required): The email address of the user.
                        - `name` (string): The name of the user.
                        - `role` (string): The role of the user (e.g., "user", "admin").
                        - `profile_complete` (boolean): Indicates whether the user's profile is complete.
                        - `profile_picture` (string): URL of the user's profile picture.
                        - `password` (string, required): The password for the user account.
                        - `country` (string): The user's country.
                        - `gender` (string): The user's gender.
                        - `institution` (string): The user's institution or organization.
                        - `mobile` (string): The user's mobile phone number.
                        
                        Example Request Body:
                        
                        ```json
                        {
                            "email": "user@example.com",
                            "name": "John Doe",
                            "role": "user",
                            "profile_complete": true,
                            "profile_picture": "https://example.com/profile.jpg",
                            "password": "password",
                            "country": "United States",
                            "gender": "Male",
                            "institution": "Example University",
                            "mobile": "+1234567890"
                        }


                        Responses


                        201 Created: User registered successfully.

                        Response Body:
                        json
                        Copy code
                        {
                            "msg": "User registered successfully"
                        }
                        400 Bad Request: Missing or invalid email or password.

                        Response Body:
                        json
                        Copy code
                        {
                            "msg": "Email and password are required"
                        }
                        400 Bad Request: User with the same email already exists.

                        Response Body:
                        json
                        Copy code
                        {
                            "msg": "User already exists"
                        }

Authenticate and log in a user, providing an access token

POST https://api-docs-studyhacks.onrender.com/login

                    
                        ## User Login
                        - **URL**: `/login`
                        - **Method**: `POST`
                        
                        ### Request Body
                        
                        - `email` (string, required): The email address of the user.
                        - `password` (string, required): The password for the user account.
                        
                        Example Request Body:
                        
                        ```json
                        {
                            "email": "user@example.com",
                            "password": "password"
                        }
                        
                        
                        Responses
                        
                        200 OK: Login successful. Returns the 
                        user's information and an access token.

                        Response Body:
                        json
                        Copy code
                        {
                            "msg": "Login successful",
                            "access_token": "your_access_token",
                            "user": {
                                "email": "user@example.com",
                                "name": "John Doe",
                                "role": "user",
                                "profile_complete": true,
                                "profile_picture": "https://example.com/profile.jpg",
                                "password": "password",
                                "country": "United States",
                                "gender": "Male",
                                "institution": "Example University",
                                "mobile": "+1234567890"
                            }
                        }
                        400 Bad Request: Missing or invalid email or password.

                        Response Body:
                        json
                        Copy code
                        {
                            "msg": "Email and password are required"
                        }
                        401 Unauthorized: Invalid credentials.

                        Response Body:
                        json
                        Copy code
                        {
                            "msg": "Invalid credentials"
                        }

Log out the current user

POST https://api-docs-studyhacks.onrender.com/logout

                    

                        
                        Log out the current user. Authentication and a valid JWT
                         (JSON Web Token) access token are required to access this endpoint.

                        Request
                        Method: POST
                        Authentication: Requires a valid JWT access token.
                        Authorization Header: Include the JWT
                        access token in the Authorization header.

                        Response
                        Status Codes:
                        200 OK - Successful user logout.
                        401 Unauthorized - Authentication failure.
                        Successful Logout (Status Code: 200)
                        Upon successful user logout, the server responds 
                        with a JSON message confirming the 
                    
                    
                        Response:
                        {
                            "msg": "Logged out successfully"
                        }

GET https://api-docs-studyhacks.onrender.com/users/<user_id>

Get user details by user ID.

                    
                        Retrieve user details by user ID. Authentication and a
                         valid JWT (JSON Web Token) access token are required to 
                         access this endpoint.
                        
                        Request

                        Method: GET
                        URL Parameter:
                        id (string): The unique identifier of the user.
                        Authentication: Requires a valid JWT access token.
                        Authorization Header: Include the JWT access 
                        token in the Authorization header.

                        Response
                        Status Codes:
                        200 OK - Successful retrieval of user details.
                        404 Not Found - User with the specified ID does not exist.
                        401 Unauthorized - Authentication failure.
                        Successful Retrieval (Status Code: 200)
                        Upon successful retrieval of user details, the server 
                        responds with a JSON object containing the user's information.
                    
                    
                        Response:
                        {
                            "user_id": "user_id",
                            "name": "John Doe",
                            "email": "user@example.com",
                            "role": "user",
                            "profile_complete": true,
                            "profile_picture": "https://example.com/photo.jpg"
                        }

Summary Processing Endpoints

Generate a summary from the provided text data.

Generate a summary from the provided text data.

class="method">POST https://api-docs-studyhacks.onrender.com/summary

This endpoint generates a summary from text data provided by the user. It requires the user to be authenticated with a valid access token.

                    
                        Request Header:
                        {
                            "Authorization": "Bearer your_access_token"
                        }
                        Request Body:
                        {
                            "text_data": "The text data from which 
                            you want to create a summary."
                        }
                        Response:
                        {
                            "msg": "Summary created successfully",
                            "summary": "The generated summary text."
                        }

Generate a summary from a PDF document using its unique identifier.

GET https://api-docs-studyhacks.onrender.com/summary/<pdf_id>

Generate a summary from a PDF document using its unique identifier.

This endpoint generates a summary from a PDF document specified by its unique identifier (`pdf_id`). It requires the user to be authenticated with a valid access token.

                    
                        Request Header:
                        {
                            "Authorization": "Bearer your_access_token"
                        }
                        Response:
                        {
                            "msg": "Summary created successfully",
                            "summary": "The generated summary text."
